En esta practica aprendimos sobre el Modelo OSI y el Modelo TCP/IP

Modelo OSI

                            


El modelo de Interconexión de Sistemas Abiertos (OSI, por 
sus siglas en inglés) es un marco conceptual utilizado para 
entender y diseñar la arquitectura de las redes de computadoras. 

Se divide en siete capas, cada una de las cuales realiza funciones 
específicas y se comunica con las capas adyacentes. Este modelo 
fue desarrollado por la Organización Internacional de Normalización 
(ISO) y es descrito en detalle en la quinta edición del libro 
"Redes de Computadora" por Andrew S. Tanenbaum. 



Las Siete Capas del Modelo OSI 



Capa Física: 

Función: Define las características físicas del hardware de red, 
incluidos los medios de transmisión, las señales eléctricas, las 
tasas de bits y las interfaces físicas. Se encarga de la transmisión 
de bits brutos a través de un canal de comunicación. 


Ejemplos: Cables de red, hubs, repetidores. 



Capa de Enlace de Datos: 

Función: Asegura una transferencia de datos libre de errores entre 
dos nodos adyacentes, manejando la detección y corrección de errores, 
así como el control de flujo. Se divide en dos subcapas: Control de 
Enlace Lógico (LLC) y Control de Acceso al Medio (MAC). 


Ejemplos: Switches, protocolos Ethernet, Wi-Fi (802.11). 



Capa de Red: 

Función: Maneja el direccionamiento, la asignación de rutas y el 
encaminamiento de paquetes entre redes. Proporciona funciones de 
conmutación de paquetes y determina la ruta óptima para el envío de 
datos. 


Ejemplos: Routers, protocolos IP (IPv4, IPv6). 



Capa de Transporte: 

Función: Proporciona una transferencia de datos confiable y 
transparente entre los puntos finales de una red. Implementa 
control de flujo, multiplexación y la corrección de errores 
de extremo a extremo. 


Ejemplos: Protocolos TCP (Transmission Control Protocol), 
UDP (User Datagram Protocol). 



Capa de Sesión: 

Función: Establece, administra y termina las sesiones entre 
aplicaciones. Gestiona el diálogo y controla la sincronización 
de datos, ofreciendo puntos de verificación y recuperación de fallos. 


Ejemplos: RPC (Remote Procedure Call), protocolos de inicio de 
sesión en red. 



Capa de Presentación: 

Función: Traduce los datos entre el formato de red y el formato 
que puede entender la aplicación. Se encarga de la codificación, 
la compresión y el cifrado de datos. 


Ejemplos: Protocolos SSL/TLS (Secure Sockets Layer/Transport Layer 
Security), formatos de datos como JPEG, ASCII, EBCDIC. 



Capa de Aplicación: 

Función: Proporciona servicios de red directamente a las aplicaciones 
del usuario final. Es la capa más cercana al usuario y facilita la 
interacción con el software de aplicación. 


Ejemplos: Protocolos HTTP (Hypertext Transfer Protocol), FTP 
(File Transfer Protocol), SMTP (Simple Mail Transfer Protocol). 



Interacción entre Capas 


Cada capa del modelo OSI interactúa solo con las capas directamente 
adyacentes a ella. Un cambio en una capa generalmente no afecta 
las otras, lo que hace que el modelo OSI sea modular y flexible. 
La comunicación entre capas se realiza a través de interfaces bien 
definidas, lo que permite que diferentes tecnologías y protocolos 
sean intercambiables y que puedan trabajar juntos de manera eficiente. 



Beneficios del Modelo OSI 


Estándar Abierto: Facilita la interoperabilidad entre diferentes 
fabricantes y tecnologías. 


Modularidad: Permite el desarrollo y la actualización 
independiente de cada capa. 


Diagnóstico de Problemas: Facilita la identificación y resolución 
de problemas al aislar fallos en capas específicas. 


Educación: Proporciona un marco estructurado para el aprendizaje y 
la enseñanza de las redes de computadoras. 



En conclusión, el modelo OSI es esencial para entender cómo se diseñan, 
implementan y gestionan las redes de computadoras. Cada capa tiene un 
papel específico y todas juntas permiten la comunicación eficiente y 
efectiva entre dispositivos de red.

Modelo TCP/IP

 
El modelo TCP/IP, también conocido como el modelo de Internet, 
es un marco conceptual utilizado para la comunicación en redes 
de computadoras. A diferencia del modelo OSI de siete capas, el 
modelo TCP/IP tiene una estructura de cuatro capas. Este modelo es 
fundamental para el funcionamiento de Internet y es descrito en 
detalle en la quinta edición del libro "Redes de Computadora" 
por Andrew S. Tanenbaum.

Las Cuatro Capas del Modelo TCP/IP

1.Capa de Acceso a la Red (o Capa de Enlace)

Función: Maneja la transmisión de datos a través del enlace físico 
entre dos dispositivos en la misma red. Se encarga del acceso al 
medio, el direccionamiento físico (MAC), el control de errores y la 
sincronización de trama.

Ejemplos: Ethernet, Wi-Fi, ARP (Address Resolution Protocol).

2.Capa de Internet

Función: Se ocupa del direccionamiento lógico y del encaminamiento 
de paquetes entre redes diferentes. Esta capa define el formato de 
los paquetes y el protocolo de enrutamiento para la entrega de datos.

Ejemplos: IP (Internet Protocol), ICMP (Internet Control Message 
Protocol), IGMP (Internet Group Management Protocol).

Capa de Transporte

Función: Proporciona una comunicación de extremo a extremo, garantizando 
la entrega correcta y ordenada de los datos. Implementa control de flujo, 
corrección de errores y segmentación de datos.

Ejemplos: TCP (Transmission Control Protocol), UDP (User Datagram Protocol).

Capa de Aplicación

Función: Contiene los protocolos que los programas de las aplicaciones 
utilizan para comunicarse a través de la red. Esta capa es la más cercana 
al usuario final y proporciona servicios de red directamente a las 
aplicaciones.

Ejemplos: HTTP (Hypertext Transfer Protocol), FTP (File Transfer Protocol), 
SMTP (Simple Mail Transfer Protocol), DNS (Domain Name System).

Interacción entre Capas

Al igual que en el modelo OSI, cada capa del modelo TCP/IP interactúa solo 
con las capas adyacentes. Cada capa agrega su propia información de 
encabezado a los datos que recibe antes de pasarlos a la siguiente capa. 
Este proceso se llama "encapsulación". La modularidad del modelo TCP/IP 
permite que las tecnologías y protocolos dentro de cada capa puedan ser 
desarrollados y actualizados de forma independiente.

Comparación con el Modelo OSI

Capa de Acceso a la Red (TCP/IP) vs. Capa Física y Capa de Enlace de Datos 
(OSI):

La capa de acceso a la red en TCP/IP combina las funciones de las capas 
física y de enlace de datos del modelo OSI.

Capa de Internet (TCP/IP) vs. Capa de Red (OSI):

Ambas capas tienen funciones similares, encargándose del direccionamiento 
y el enrutamiento de paquetes. Sin embargo, el modelo TCP/IP se centra 
específicamente en el protocolo IP.

Capa de Transporte (TCP/IP) vs. Capa de Transporte (OSI):

Las funciones son casi idénticas, incluyendo el control de flujo, la 
segmentación y la corrección de errores. TCP y UDP son los principales 
protocolos de transporte en el modelo TCP/IP.

Capa de Aplicación (TCP/IP) vs. Capas de Sesión, Presentación y Aplicación 
(OSI):

La capa de aplicación del modelo TCP/IP combina las funciones de las capas 
de sesión, presentación y aplicación del modelo OSI, proporcionando un 
marco más simplificado y menos estricto.

Beneficios del Modelo TCP/IP

Simplicidad y Eficiencia: Menos capas hacen que el modelo sea más fácil 
de implementar y entender.

Flexibilidad y Robustez: Adaptable a diversas tecnologías de red y 
entornos de hardware.

Compatibilidad: Es el modelo utilizado por Internet, lo que asegura la 
compatibilidad global y el soporte extendido. 

Interoperabilidad: Facilita la comunicación entre diferentes sistemas 
y tecnologías de red.

En conclusión, el modelo TCP/IP es fundamental para la estructura y el 
funcionamiento de Internet. Proporciona un marco flexible y eficiente para 
la comunicación en red, que ha demostrado ser escalable y robusto a lo 
largo del tiempo. Su simplicidad y la agrupación de funciones en capas 
manejables hacen que sea una opción preferida para el diseño y la 
implementación de redes de computadoras.
